## 五、真实差距分析

### 与商用量化系统的GAP

| 差距 | 影响 | 解决难度 |
|------|------|---------|
| **Tick数据缺失** | 最致命：Level2数据只有5档，非真实逐笔 | 高（需商业数据源） |
| **策略单一** | 只有RSI均值回归，趋势/动量缺失 | 中（框架已就绪） |
| **历史数据不足** | 回测仅用沪深300ETF，其他A股/港股未验证 | 高（需数据采购） |
| **无真实成交验证** | PaperTrade和真实订单执行差异未量化 | 中（需接入券商） |
| **协方差矩阵估算** | BL用简化相关矩阵，非真实市场协方差 | 高（需更高频数据） |
| **VaR/CVaR缺失** | 组合层面风险仅用简单暴露度，无蒙特卡洛 | 高 |

### 最关键的三个问题

**1. 数据是核心瓶颈**
```
当前: 日线 + 5档L2 + 随机游走模拟
商用: Tick级 + Level2逐笔 + 订单簿重构
差距: 约10倍的信号质量差距
```

**2. 策略需要从1个变成5个**
```
当前: RSI均值回归（1个alpha）
商用: 均值回归 + 趋势跟踪 + 动量 + 套利 + 宏观对冲（5+个）
框架已支持，但只有1个策略在跑
```

**3. 回测过拟合风险高**
```
RSI参数在2013-2026验证，Sharpe=0.467
但只有1个WFA窗口（2018-2019）
统计意义弱：可能参数过拟合
```
